# gemini-3-flash-preview_2026-03-03_0652_59

The Gemini CLI stores its conversations (sessions) as JSON files in a project-specific directory within your home folder.

### **Storage Locations**
Conversations are stored based on your operating system and the specific project you are working in:

*   **Linux/macOS:** `~/.gemini/tmp/<project_hash>/chats/`
*   **Windows:** `C:\Users\<YourUsername>\.gemini\tmp\<project_hash>\chats\`

The `<project_hash>` is a unique identifier generated by the CLI based on your project's root directory. This ensures that conversation histories are scoped to the specific project you are working on.

Additionally:
*   **Manual Checkpoints:** If you use the `/chat save <tag>` command, the state is saved in the same `chats/` directory.
*   **Automatic Checkpoints:** If [Checkpointing](cli/checkpointing.md) is enabled, snapshots taken before file modifications are stored in `~/.gemini/tmp/<project_hash>/checkpoints`.
*   **Shadow Git History:** Snapshots of your project files for the "Rewind" feature are stored in `~/.gemini/history/<project_hash>`.

---

### **Finding the Current Conversation**
You can find and access the current or most recent conversation using several methods:

#### **1. Within the Interactive CLI**
*   **Session Browser:** Type **`/resume`** to open an interactive browser where you can search, filter, and select from all saved sessions for the current project.
*   **Session Stats:** Type **`/stats session`** to see details and metadata about the currently active session.
*   **Export:** Type **`/chat share [filename]`** (e.g., `/chat share my_chat.md`) to export the current conversation history to a file.

#### **2. From the Command Line**
*   **List Sessions:** Run `gemini --list-sessions` to see a numbered list of all available sessions for the current directory/project.
*   **Resume Latest:** Run **`gemini --resume`** (or **`gemini -r`**) to immediately load the most recently active session.

#### **3. In the File System**
To find the raw data file for the current session:
1.  Navigate to the `chats/` directory for your project (e.g., `~/.gemini/tmp/<project_hash>/chats/`).
2.  The current or most recent session is typically the **most recently modified `.json` file** in that folder.

To access Gemini VS Code plugin chats programmatically, you can follow these steps to locate the relevant session files:

### **1. Identify the Project Hash/Slug**
The Gemini CLI maps your project's root directory to a unique "slug" or "hash." You can find this mapping in the global Gemini configuration file:
-   **File Path:** `~/.gemini/projects.json`
-   **Structure:**
    ```json
    {
      "projects": {
        "/path/to/your/project": "project-slug"
      }
    }
    ```

### **2. Locate the Chat Directory**
Once you have the project slug (e.g., `kato`), the chats are stored in a `chats` subfolder within the project's temporary directory:
-   **Directory Path:** `~/.gemini/tmp/<project-slug>/chats/`

### **3. Read Session Files**
The chats are stored as individual JSON files named `session-<timestamp>-<id>.json`. 
-   **File Format:** Each file contains a single JSON object.
-   **Key Fields:**
    -   `sessionId`: The unique identifier for the session.
    -   `lastUpdated` / `updatedAt`: ISO timestamp of the last message.
    -   `messages`: An array of message objects containing `role`, `content`, and `displayContent`.

### **Example (Node.js snippet):**
```javascript
const fs = require('fs');
const path = require('path');

const projectPath = '/home/djradon/hub/spectacular-voyage/kato';
const projectsMapping = JSON.parse(fs.readFileSync(path.join(process.env.HOME, '.gemini/projects.json'), 'utf8'));
const projectSlug = projectsMapping.projects[projectPath];

if (projectSlug) {
  const chatDir = path.join(process.env.HOME, '.gemini/tmp', projectSlug, 'chats');
  const files = fs.readdirSync(chatDir).filter(f => f.endsWith('.json'));
  
  // Read the most recent session
  const latestFile = files.sort().reverse()[0];
  const chatData = JSON.parse(fs.readFileSync(path.join(chatDir, latestFile), 'utf8'));
  console.log(`Last message:`, chatData.messages.slice(-1)[0].content);
}
```

This approach allows you to reliably locate and parse Gemini conversations for any project managed by the extension or CLI.
